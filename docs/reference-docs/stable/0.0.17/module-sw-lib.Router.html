Router

Adds a friendly API on top of the router from the sw-routing module.

Examples

How to define a simple route with caching strategy.

goog.swlib.router.registerRoute('/about', goog.swlib.cacheFirst());

How to define a simple route with custom caching strategy.

self.goog.swlib.router.registerRoute('/about', (args) => {
  // The requested URL
  console.log(args.url);

  // The FetchEvent to handle
  console.log(args.event);

  // The parameters from the matching route (Commonly
  // used with Regex / Express routes).
  console.log(args.params);

  // Return a promise that resolves with a Response.
  return fetch(args.url);
}));

Constructor

new Router(revisionedCacheName)

Constructs a light wrapper on top of the underlying Router.

Parameters
revisionedCacheName String

The cache name used for entries cached via cacheRevisionedAssets().

Instance Methods

this.registerNavigationRoute(url, options)

A shortcut used to register a module:sw-routing.NavigationRoute instance that will respond to navigation requests using a cache entry for url.

This is useful when following the App Shell pattern, in which the previously cached shell is returned for all navigations.

The url value should correspond to an entry that's already in the cache, perhaps a URL that is managed by module:sw-lib.SWLib#cacheRevisionedAssets. Using a URL that isn't already cached will lead to failed navigations.

Parameters
url String

The URL of the already cached HTML resource.
options Object (Optional)
Properties
blacklist Array.<RegExp> (Optional)

Defaults to an empty blacklist.
whitelist Array.<RegExp> (Optional)

Defaults to [/./], which will match all request URLs.
cacheName String (Optional)

The name of the cache which contains the cached response for url. Defaults to the name of the cache used by cacheRevisionedAssets().

this.registerRoute(capture, handler)

Parameters
capture String | Regex | Route

The capture for a route can be one of three types.

  1. It can be an Express style route, like: '/example/:anything/route/' The only gotcha with this is that it will only capture URL's on your origin.
  2. A regex that will be tested against request URL's.
  3. A Route instance.
handler function | Handler

Called when the route is caught by the capture criteria. The handler argument is ignored if you pass in a Route object, otherwise it's required. If required, provide a function or a runtime caching strategy.
